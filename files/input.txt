A JavaScript Promise is an object that represents the eventual result of an asynchronous operation—either a value that becomes available later or an error if something goes wrong.
Many operations in JavaScript don’t finish immediately, for example:
Fetching data from a server
Reading a file
Waiting for a timer
Doing work in the background
Promises give us a clean, structured way to handle results that arrive later, without deeply nested callbacks
A Promise is always in one of these states:
Pending – the operation hasn’t finished yet
Fulfilled – the operation completed successfully (you get a value)
Rejected – the operation failed (you get an error)
Once a promise is fulfilled or rejected, it is settled and can’t change state.
